# Contributing to Converge

Converge contributions should improve decision quality, consistency, or conformance with the framework.

## Principles

- Prefer precise additions over broad rewrites.
- Keep every new document or clause scoped to one responsibility.
- Support claims with reasoning, examples, or explicit criteria.
- Avoid duplicating content that already lives in another canonical document.

## Contribution Types

- New specialist definitions for the council.
- New domain packs for adaptive guidance.
- New engine behavior or scoring rules.
- New gates, templates, examples, or conformance fixtures.
- Corrections that improve clarity without changing intent.

## Review Expectations

Any change should explain:

- Why it belongs in Converge.
- What decision problem it solves.
- How it interacts with existing framework concepts.
- Whether it introduces ambiguity, redundancy, or hidden coupling.

## Pull Request Checklist

- The change has a clear purpose.
- The change does not duplicate another document.
- The terminology matches the glossary.
- The content is internally consistent.
- The new artifact can be referenced from the spec or README when appropriate.
