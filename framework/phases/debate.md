# Debate

## Purpose

Compare candidate approaches using specialist judgment and explicit evaluation criteria. The output of Debate is clarity about which option is strongest and why, not unanimity of opinion. Disagreement that is recorded and reasoned is more valuable than agreement that is unexamined.

## Entry Condition

The problem framing has survived Challenge. A set of candidate approaches has been identified, either explicitly during Investigate or as a product of the Challenge process.

## Process

1. List all candidate approaches. Each must be stated clearly enough that a council member who preferred a different option could still describe this one accurately.
2. Define the evaluation criteria before scoring begins. Criteria must derive from the success conditions established in Understand and the hard constraints recorded there. Adding criteria mid-debate requires an explicit decision.
3. Each specialist evaluates each option from their domain perspective: what this option makes easier, what it makes harder, what risks it introduces, and what evidence supports the evaluation.
4. Surface explicit tradeoffs. Every meaningful option gives something up. An option that appears to have no tradeoffs has not been evaluated carefully enough.
5. Record disagreements with the reasoning on both sides. A disagreement that is clearly articulated is useful input. A disagreement that is suppressed is a future incident.
6. Identify the option with the best overall tradeoff profile given the constraints and success criteria.

## Outputs

- Candidate list with a clear description of each option.
- Evaluation criteria derived from Understand.
- Per-specialist evaluation for each candidate.
- Explicit tradeoff summary for each option.
- Recorded disagreements with the reasoning on each side.
- Preliminary preferred option with the reasoning that supports it.

## Exit Condition

Every candidate has been evaluated against every criterion. The preferred option is identified. Every council member understands why the preferred option was chosen, even if not all of them agree with the choice.

## Failure Signals

- The preferred option was decided before Debate began and the process is being used to justify it.
- Evaluation criteria are vague ("better architecture", "more scalable").
- Tradeoffs are described for weak options but omitted for the preferred option.
- Specialist input is collected but not incorporated into the evaluation.
- Debate produces more options than it started with and no elimination.

## Interaction with Subsequent Phases

The output of Debate feeds directly into Converge. If the preferred option is clear and disagreement is minimal, Converge is fast. If disagreements are deep or criteria conflict, Converge must resolve them explicitly before the cycle can advance to Decide.
