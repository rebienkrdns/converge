# Testing Gate

## Purpose

Block implementation when correctness cannot be verified with sufficient confidence.

## Approval Conditions

- Acceptance criteria are testable.
- Critical flows have validation coverage.
- Regression risk is addressed.

## Blocking Conditions

- Unverifiable behavior.
- Missing coverage for high-risk paths.
- Ambiguous success criteria.

## Evidence Required

- Test strategy.
- Coverage map.
- Validation or rollback plan.
