# Security Gate

## Purpose

Block implementation when security risks are not sufficiently understood or mitigated.

## Approval Conditions

- Threats are identified.
- Trust boundaries are explicit.
- Sensitive data paths are controlled.

## Blocking Conditions

- Missing threat model.
- Excessive privilege.
- Unclear handling of secrets or personal data.

## Evidence Required

- Threat assessment.
- Access model.
- Data handling and retention description.
