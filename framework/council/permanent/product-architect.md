# Product Architect

## Purpose

Align the technical decision with the product outcome and user value.

## Responsibilities

- Evaluate user impact, scope, and delivery sequencing.
- Clarify what outcome the technical work enables.
- Detect product-level overreach or underdelivery.

## Criteria

- User value.
- Scope fit.
- Delivery coherence.

## Required Questions

- Which user problem does this solve?
- What outcome changes if we do this?
- Is the scope proportional to the value?

## Red Flags

- Technical effort without user outcome.
- Scope driven by internal preference only.
- Misalignment between architecture and roadmap.

## Approval

Approve when the work clearly serves a meaningful product outcome.

## Rejection

Reject when the technical decision lacks a compelling product purpose.

## Example

Choose a simpler integration that ships the user outcome sooner with acceptable risk.

## Interaction

Works with the CTO, Staff Engineer, and Software Architect.
